Executive Summary

The NHAI Retroreflectivity AI Pipeline is a cutting-edge computer vision system designed to automate the assessment and monitoring of retroreflectivity for highway infrastructure elements. By leveraging advanced machine learning models and physics-based calculations, the system replaces manual reflectometer inspections with scalable, real-time compliance monitoring across NHAI's extensive highway network.

Project Overview

Objectives

  • Predict retroreflectivity in standard units (mcd/m²/lux)
  • Verify real-time compliance against IRC 35 (pavement markings) and IRC 67 (traffic signs) standards
  • Forecast degradation and recommend maintenance schedules 3-6 months in advance
  • Replace manual reflectometer checks with automated AI-driven solution
  • Provide geotagged insights to field operations via interactive dashboard

Key Innovations

  • Physics-Informed ML: Combines computer vision with optical physics
  • Condition Normalization: Adapts to 6 environmental conditions
  • Temporal Aggregation: Groups detections by 100m highway segments
  • Degradation Forecasting: Predictive maintenance scheduling
  • GPS Integration: Geotagging for field navigation
  • India-Specific Training: Datasets from Indian highways

AI Models & Training Results

Model A: Environmental Condition Classifier

Architecture: EfficientNet-B0 with custom classifier head

Purpose: Classifies frames into 6 environmental conditions for adaptive preprocessing

Training Details:

  • Dataset: 12,000 images (2,000 per condition)
  • Batch size: 32
  • Learning rate: 0.001
  • Epochs: 50 (currently ~20/50 complete)
  • Augmentations: Horizontal flip, brightness/contrast, Gaussian blur
  • Regularization: Dropout (0.3), weight decay (0.01)

How it Works: Takes full frame (224×224), outputs condition label + confidence. Conditions: day_clear, night_clear, fog, rain, wet_road, night_fog.

Model B: Infrastructure Element Detector (YOLOv11)

Architecture: YOLOv11 nano variant

  • Classes: road_marking (0), traffic_sign (1), road_stud (2), delineator (3)

How it Works: Processes frame resized to 1024×1024, outputs bounding boxes + class labels + confidence scores. Minimum confidence: 0.10, minimum size: 50×50 pixels.

Model C: Retroreflectivity Estimator

Architecture: Physics-based regression model

Purpose: Converts pixel brightness to actual retroreflectivity values

Training Details:

  • Dataset: Synthetic data generated from physics equations
  • Validation: Real-world reflectometer measurements
  • Parameters: Condition factors, distance corrections

How it Works: Uses inverse photometric relationships: RL = K × (L_pixel/L_max) × F_condition × F_distance, where K=600.0 (IRC 35 constant), with condition-specific correction factors.

System Architecture

10-Step Pipeline

  1. Multi-Condition Video Ingestion: Classifies frames into 6 environmental conditions
  2. Condition-Specific Preprocessing: Applies DehazeNet, Retinex enhancement, glare suppression
  3. Element Detection & Segmentation: YOLOv11 + SAM2 for pixel-precise infrastructure detection
  4. Physics-Informed Luminance Regression: Converts pixel brightness to retroreflectivity (mcd/m²/lux)
  5. Instant IRC Compliance Checker: Real-time flagging of non-compliant elements with GPS
  6. Temporal Aggregation Module: Groups readings by 100m highway segments
  7. Degradation Prediction Model: XGBoost for predicting maintenance timelines
  8. Weather-Adaptive Retraining Loop: Incremental model retraining
  9. FastAPI Backend & Dashboard: REST API with PostGIS spatial database
  10. Simulation & Demo Layer: End-to-end demonstration on public datasets

Architecture Summary

The system flows from raw video and GPS input through condition classification, adaptive preprocessing, YOLO detection, physics-based retroreflectivity estimation, and then into compliance reporting and geospatial visualization. This complete pipeline is implemented in Python with FastAPI backend and React dashboard for full project delivery.
